## 〇、方案总览：三层架构

> 最新口径：策略不是固定80%仓位，也不是机械保留20%现金。老师认可80%仓位为合规底线后，本方案采用**动态目标仓位**：每日目标仓位在 `[80%, 100%]` 区间内自适应变化，并默认给80%底线留2%执行缓冲。市场波动高、模型分数不够集中时靠近底线；市场波动低、Top股票分数优势明显时提高到90%-100%。80%只是红线底线，不是写死的数据点。

### 1.2 量价因子（约15个特征）

这是最核心的一组特征，直接从OHLCV数据衍生：

```
# 收益率类
ret_1d   = close(t) / close(t-1) - 1        # 1日收益率
ret_5d   = close(t) / close(t-5) - 1        # 5日收益率
ret_20d  = close(t) / close(t-20) - 1       # 20日收益率

# 波动率类
volatility_5d  = std(ret_1d, window=5)       # 5日波动率
volatility_20d = std(ret_1d, window=20)      # 20日波动率
vol_ratio      = volatility_5d / volatility_20d  # 波动率比（趋势强度信号）

# 量价关系类
volume_ratio_5d  = vol(t) / mean(vol, 5d)    # 量比（5日）
volume_ratio_20d = vol(t) / mean(vol, 20d)   # 量比（20日）
amount_per_vol   = amount(t) / vol(t)         # 均价（成交额/成交量）
turnover_change  = volume_ratio_5d / volume_ratio_20d  # 换手变化

# K线形态类
upper_shadow = (high - max(open, close)) / (high - low + 1e-8)  # 上影线比
lower_shadow = (min(open, close) - low) / (high - low + 1e-8)   # 下影线比
body_ratio   = abs(close - open) / (high - low + 1e-8)          # 实体比
high_low_ratio = (high - low) / close                            # 振幅
```

### 1.4 截面排序特征（71+个特征）—— ⭐创新点之一

传统方法只看个股自身的时间序列。截面特征是在每个交易日把所有股票横向排名，捕捉"相对位置"信息：

```python
# 每个交易日，对全部约5000只股票做以下排序
# 基础特征排名（约30个）
rank_ret_1d    = rank(ret_1d)   / N   # 1日收益率分位数
rank_ret_5d     = rank(ret_5d)   / N   # 5日收益率分位数
rank_ret_20d    = rank(ret_20d)  / N   # 20日收益率分位数
rank_vol_ratio  = rank(volume_ratio_5d) / N  # 量比分位数
rank_volatility = rank(volatility_20d) / N  # 波动率分位数
rank_amount     = rank(amount)   / N   # 成交额分位数
rank_turnover   = rank(turnover) / N   # 换手率分位数
# ... 共71个基础特征参与截面排名

# 技术指标排名
rank_rsi_14, rank_macd_hist, rank_kdj_k, rank_adx, rank_supertrend ...

# 资金流排名
rank_net_mf_amount, rank_mf_ratio, rank_mf_acceleration ...

# 行业排名（行业内相对位置）
ind_rank_ret_5d, ind_rank_volatility, ind_rank_mf_ratio ...

# 风格排名
rank_size_neutral_ret, rank_value_score, rank_trend_persistence ...
```

**为什么截面特征有效？** 金融市场中的alpha信号是"相对收益"而非"绝对收益"。一只股票涨了3%，如果全市场平均涨5%，它其实是弱势的。截面排名天然编码了这种相对信息。

### 1.8 标准化方案（防止数据泄露）

**关键原则：** 只能用当前时间窗口及之前的数据计算统计量。

```python
# 方案：滚动窗口Z-Score标准化
# 对于每个交易日t，用过去rolling_window天的数据计算均值和标准差
def rolling_normalize(feature_series, window=60):
    rolling_mean = feature_series.rolling(window).mean()
    rolling_std  = feature_series.rolling(window).std()
    return (feature_series - rolling_mean) / (rolling_std + 1e-8)
```

截面特征（rank类）本身已经是[0,1]分布，不需要额外标准化。

### 1.9 标签构造

```python
# 预测目标：未来n日收益率
# 推荐 n=1（次日收益率），也可尝试 n=3 或 n=5
label = close(t+n) / close(t) - 1

# 可选：对label做截面排名转换（rank label）
# 好处：消除市场整体涨跌的影响，让模型专注学习相对排序
label_rank = rank(label) / N  # [0, 1]
```

**推荐使用rank label。** 理由：我们最终的交易策略是"选排名最高的股票买入"，所以模型只需要学会排序，不需要精确预测收益率数值。用rank label训练IC通常更高。

## 二、模型架构设计

### 2.1 通道A：GRU-Transformer 混合模型（主力模型）

**设计动机：** 纯Transformer在短序列上容易过拟合，纯GRU缺乏全局注意力。混合架构取两者之长。

```
输入: (batch, seq_len=20, features=200+)
        │
        ▼
┌─────────────────────────┐
│  Input Projection       │  Linear(200+, d_model=128)
│  + Positional Encoding  │  可学习位置编码
└────────────┬────────────┘
             ▼
┌─────────────────────────┐
│  Bi-GRU Layer (1层)     │  捕捉局部时序依赖
│  hidden_size = 64       │  输出维度 = 128 (双向拼接)
└────────────┬────────────┘
             ▼
┌─────────────────────────┐
│  Transformer Encoder    │  2层, 4头注意力
│  d_model=128, d_ff=256  │  捕捉全局依赖关系
│  dropout=0.1            │
└────────────┬────────────┘
             ▼
┌─────────────────────────┐
│  Temporal Attention Pool │  ⭐ 可学习的时间步加权
│  不是简单取最后一步       │  而是attention加权聚合
└────────────┬────────────┘
             ▼
┌─────────────────────────┐
│  Prediction Head        │
│  Linear(128, 64) + ReLU │
│  Dropout(0.3)           │
│  Linear(64, 1)          │  输出：预测分数
└─────────────────────────┘
```

**Temporal Attention Pooling（⭐创新点之二）：**

```python
class TemporalAttentionPool(nn.Module):
    """不是取最后一个时间步，而是让模型自己学习每个时间步的重要性权重"""
    def __init__(self, d_model):
        super().__init__()
        self.attention = nn.Sequential(
            nn.Linear(d_model, d_model // 2),
            nn.Tanh(),
            nn.Linear(d_model // 2, 1)
        )

    def forward(self, x):
        # x: (batch, seq_len, d_model)
        attn_weights = self.attention(x)          # (batch, seq_len, 1)
        attn_weights = F.softmax(attn_weights, dim=1)
        output = (x * attn_weights).sum(dim=1)    # (batch, d_model)
        return output, attn_weights  # 返回权重用于可视化！
```

**为什么返回attention weights？** 可以在报告中画出"模型认为近20天中哪几天最重要"的热力图，这是一个很强的可解释性图表，答辩时非常加分。

### 2.2 通道B：LightGBM 截面排序模型（辅助模型）

**设计动机：** 树模型擅长捕捉非线性特征交互，而且不受时序长度限制。它在截面排序任务上天然强势。

```python
import lightgbm as lgb

# 输入：不用滑动窗口，直接用当天的特征向量
# X: (num_stocks, num_features=200+) —— 当天所有股票的特征
# y: (num_stocks,) —— 次日收益率rank

params = {
    'objective': 'lambdarank',    # ⭐ 排序学习目标，不是回归
    'metric': 'ndcg',
    'ndcg_eval_at': [10, 20, 30], # 评估top-10/20/30的排序质量
    'num_leaves': 63,
    'learning_rate': 0.05,
    'feature_fraction': 0.8,
    'bagging_fraction': 0.8,
    'bagging_freq': 5,
    'verbose': -1
}

# 注意：LambdaRank需要group信息（每个query包含多少候选）
# 这里每个交易日就是一个query，包含约5000只股票
train_data = lgb.Dataset(X_train, y_train, group=group_sizes)
```

**为什么用LambdaRank而不是普通回归？** 我们的最终目标是"选出排名靠前的股票"，LambdaRank直接优化排序质量（NDCG），比MSE回归更贴合任务目标。这也是一个创新点。

### 2.3 双通道融合策略（⭐创新点之三）

```python
# 两个模型分别输出每只股票的预测分数
score_transformer = model_transformer.predict(X_seq)    # 时序模型分数
score_lgbm        = model_lgbm.predict(X_cross)         # 截面模型分数

# 先各自做截面rank标准化（消除尺度差异）
rank_transformer = cross_section_rank(score_transformer)  # [0, 1]

# 加权融合

final_score = alpha * rank_transformer + (1 - alpha) * rank_lgbm

# alpha的选择方法：在验证集上grid search
# for alpha in [0.3, 0.4, 0.5, 0.6, 0.7]:
#     计算融合后的IC和ICIR，选最优
```

**融合的价值：** 时序模型擅长捕捉"趋势延续"（momentum），树模型擅长捕捉"均值回归"和"因子暴露"。两者的预测误差往往不相关，融合后IC通常比单模型提升10-20%。

### 2.4 训练细节

**损失函数：**

```python
# 方案1：IC Loss（直接优化IC值）⭐推荐
def ic_loss(pred, target):
    """最大化预测值与真实值的Pearson相关系数"""
    pred = pred - pred.mean()
    target = target - target.mean()
    ic = (pred * target).sum() / (pred.norm() * target.norm() + 1e-8)
    return -ic  # 负号因为要最小化loss

# 方案2：MSE + Rank Loss 联合
def combined_loss(pred, target, lambda_rank=0.1):
    mse = F.mse_loss(pred, target)
    rank = rank_loss(pred, target)  # pairwise ranking loss
    return mse + lambda_rank * rank
```

**推荐IC Loss。** 理由同标签构造：我们关心的是排序能力而非精确预测，IC Loss直接优化这个目标。

```

```

## 三、交易策略设计

### 3.1 基础策略增强版

在作业推荐的top-k策略基础上，加入三个改进：

#### 改进1：波动率自适应持仓数量

```python
# 市场波动大时持仓分散（降低风险），波动小时持仓集中（提升收益）
def adaptive_n(market_volatility, base_n=20, min_n=10, max_n=30):
    """
    market_volatility: 沪深300近20日波动率
    高波动 → 持更多股票分散风险
    低波动 → 持更少股票集中收益
    """
    vol_percentile = rank_in_history(market_volatility, lookback=250)
    if vol_percentile > 0.8:   # 高波动环境
        return max_n
    elif vol_percentile < 0.2: # 低波动环境
        return min_n
    else:
        return base_n
```

#### 改进2：动量确认过滤器

```python
def momentum_filter(candidates, top_k=30):
    """
    不是只看模型分数，还要求入选股票的近期走势不能太差
    防止模型选出正在下跌途中的"价值陷阱"
    """
    filtered = []
    for stock in candidates:
        score = stock['model_score']
        # 额外条件：5日收益率不能排在全市场后20%
        if stock['rank_ret_5d'] > 0.2:
            filtered.append(stock)

    # 从filtered中按model_score选top_k
    return sorted(filtered, key=lambda x: x['model_score'], reverse=True)[:top_k]
```

#### 改进3：调仓比例与置信度挂钩

```python
def smart_rebalance(current_holdings, new_scores, base_k=3):
    """
    不是固定每天换k只，而是根据分数差异决定换多少
    如果当前持仓分数和候选股票分数差异不大，少换（减少交易摩擦）
    如果差异很大，多换
    """
    # 当前持仓中得分最低的股票分数
    worst_holding_score = sorted(current_holdings, key=lambda x: x['score'])[0]['score']
    # 候选池中得分最高的非持仓股票分数
    best_candidate_score = get_best_non_holding(new_scores)

    score_gap = best_candidate_score - worst_holding_score

    if score_gap > threshold_high:
        k = base_k + 2   # 差距大，多换
    elif score_gap < threshold_low:
        k = max(1, base_k - 1)  # 差距小，少换
    else:
        k = base_k

    return k
```

#### 改进4：动态目标仓位（80%为底线，不是固定点）

```python
def dynamic_position_ratio(score_confidence, market_vol_pct):
    """
    score_confidence: 当日Top股票相对全市场的分数优势，越高说明模型越有把握
    market_vol_pct: 市场20日波动率在历史窗口中的分位数，越高说明风险越大
    """
    floor = 0.80     # 老师认可的满仓合规底线
    min_pos = 0.82   # 执行目标下限：给手续费和价格波动留缓冲
    base_pos = 0.90
    max_pos = 1.00

    vol_score = 1 - market_vol_pct  # 低波动更适合加仓
    target = (
        base_pos
        + 0.08 * (score_confidence - 0.5)
        + 0.12 * (vol_score - 0.5)
    )
    return clip(target, min_pos, max_pos)
```

**解释：**
- 80%只是组合仓位下限，用来满足“满仓”规则口径；
- 仓位会根据模型置信度和市场风险动态变化，典型区间为82%-100%，但合规检查底线仍是80%；
- 当模型Top候选分数优势强、市场波动低时，仓位接近100%；
- 当市场波动高或分数优势不明显时，仓位降到80%-90%，保留次日补仓和先买后卖空间；
- 回测输出 `target_position_ratio`、`position_ratio`、`position_floor_breach_days`，可直接证明策略不是固定80%，且没有低于底线。

### 3.2 策略参数推荐

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 持仓数n | 30 | 经调参验证的最优值，grid: [10,15,20,30,40,50] |
| 每日调仓k | 1-3 | 不要过频繁，k=1表现最优 |
| 动态仓位下限 | 80% | 合规底线，不是固定目标 |
| 执行缓冲 | 2% | 避免手续费和开收盘波动导致收盘口径低于80% |
| 动态仓位基准 | 90% | 中性状态目标仓位 |
| 动态仓位上限 | 100% | 高置信度、低波动环境下接近满仓 |
| 选股池 | 全部A股（除ST/北交所） | 按要求 |
| 建仓方式 | 新买入等额分配，组合仓位动态调整 | 简单、可解释、便于手动交易 |
| 初始资金 | 100万 | 比赛默认 |
| 市值分位 | 2%-99% | 放宽约束允许更多中小盘 |

> 说明：动态仓位的首要目的不是承诺收益一定提高，而是在不触碰满仓底线的前提下，让组合根据市场风险和模型置信度有调节空间。收益是否提升必须用同一预测文件、同一行情面板复测；答辩中应强调这是“规则合规 + 风险收益权衡”，不是固定80%仓位。
